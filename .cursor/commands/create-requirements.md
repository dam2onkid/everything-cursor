# Create Requirements Document

You are creating a **Requirements & Problem Understanding** document for a feature.

## Instructions

1. Read the template at `.cursor/templates/requirements.md`
2. Ask the user for:
   - **Feature name** (e.g., "search", "profile-screen", "listing-management")
3. Research the codebase for any existing related code, designs, or docs for this feature
4. Generate the requirements document following the template structure
5. Save to: `docs/feature/{feature-name}/requirements.md`

## Output Rules

- Follow the exact template structure from `.cursor/templates/requirements.md`
- Keep user stories specific and actionable with acceptance criteria
- Clearly define non-goals to prevent scope creep
- List all open questions that need stakeholder input

Feature: $ARGUMENTS
